37 // Class to calculate statistics using the so-called fit to half algorithm. In this
38 // algorithm, a center value is specified, and only points greater or equal or less or equal
39 // this value are included. Furthermore, each of the included points is reflected about
40 // the center value, and these virtual points are added to the included points and
41 // the union of sets of included real points and virtual points are used for computing statistics.
42 // The specified center point is therefore the mean and median of the resulting
43 // distribution, and the total number of points is exactly twice the number of real
44 // data points that are included.
45

79  // In the following group of methods, if the size of the composite dataset
80  // is smaller than
81  // <src>binningThreshholdSizeBytes</src>, the composite dataset
82  // will be (perhaps partially) sorted and persisted in memory during the
83  // call. In that case, and if <src>persistSortedArray</src> is True, this
84  // sorted array will remain in memory after the call and will be used on
85  // subsequent calls of this method when <src>binningThreshholdSizeBytes</src>
86  // is greater than the size of the composite dataset. If
87  // <src>persistSortedArray</src> is False, the sorted array will not be
88  // stored after this call completes and so any subsequent calls for which the
89  // dataset size is less than <src>binningThreshholdSizeBytes</src>, the
90  // dataset will be sorted from scratch. Values which are not included due to
91  // non-unity strides, are not included in any specified ranges, are masked,
92  // or have associated weights of zero are not considered as dataset members
93  // for quantile computations.
94  // If one has a priori information regarding
95  // the number of points (npts) and/or the minimum and maximum values of the data
96  // set, these can be supplied to improve performance. Note however, that if these
97  // values are not correct, the resulting median
98  // and/or quantile values will also not be correct (although see the following notes regarding
99  // max/min). Note that if this object has already had getStatistics()
100  // called, and the min and max were calculated, there is no need to pass these values in
101  // as they have been stored internally and used (although passing them in shouldn't hurt
102  // anything). If provided, npts, the number of points falling in the specified ranges which are
103  // not masked and have weights > 0, should be exactly correct. <src>min</src> can be less than
104  // the true minimum, and <src>max</src> can be greater than the True maximum, but for best
105  // performance, these should be as close to the actual min and max as possible.
