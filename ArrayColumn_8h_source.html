62 // <etymology>
63 // ArrayColumn<T> gives read and write access to an column in a table
64 // containing an array with data type T.
65 // </etymology>
66 
67 // <synopsis>
68 // The class ArrayColumn allows readonly access to a column
69 // containing arrays with an arbitrary data type. It can handle direct
70 // as well as indirect arrays.
71 // It is possible to get the data in an individual cell (i.e. table row);
72 // either the whole array or a slice of the array can be accessed.
73 // It is also possible to get the column as a whole if the arrays
74 // in all cells of the column have the same shape (which is always true
75 // for direct arrays). As in the case of individual cells it is possible
76 // to get the entire arrays or a slice of the arrays.
77 //
78 // A default constructor is defined to allow construction of an array
79 // of ArrayColumn objects. However, this constructs an object not
80 // referencing a column. Functions like get, etc. will fail (i.e. result
81 // in a segmentation fault) when used on such objects. The functions
82 // isNull and throwIfNull can be used to test on this.
83 // The functions attach and reference can fill in the object.
84 //
85 // The assignment operator is not defined for this class, because it was
86 // felt it would be too confusing. Instead the function reference can
87 // be used to do assignment with reference semantics. An assignment
88 // with copy semantics makes no sense for a readonly column.
89 // </synopsis>
90

92 // <li> Default constructor
93 // <li> Copy constructor
94 // <li> Assignment operator
95 // </templating>
